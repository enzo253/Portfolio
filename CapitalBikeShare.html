Capital Bikeshare Data Analysis & Prediction

This project analyzes Capital Bikeshare trip data to uncover urban mobility patterns and predict daily rider counts. Using EDA, spatial data integration, and machine learning models, it provides actionable insights into biking trends across Washington, D.C.

🛠️ Tech Stack

  • Python: Data processing and modeling using Pandas, NumPy, Scikit-Learn
  • Seaborn & Matplotlib: Exploratory data analysis and visualization of rider patterns
  • GeoPandas: Spatial data analysis with ward shapefiles for geographic insights
  • Jupyter Notebook: Interactive development environment for analysis and modeling

⚙️ Process & Methodology

The project began by merging Capital Bikeshare trip data with ward shapefiles to add a geographic dimension to the analysis. After cleaning the data and handling missing values, I performed exploratory analysis to visualize trip patterns by time, demographics, and weather factors. Multiple regression models were trained and evaluated to predict daily rider counts.

Key metrics like RMSE, MAE, and R-squared were used to select the best-performing models (Random Forest and XGBoost). Weather data was incorporated to boost prediction accuracy, providing a holistic view of urban mobility behaviors.

📈 Key Findings

  • 📊 Analyzed the distribution of bike types across all stations.
  • 👥 Compared usage patterns between members and casual riders for each bike type.
  • 📈 Measured year-over-year growth across bike types and rider categories (members vs. casuals).
  • 🤖 Achieved high predictive accuracy in forecasting daily ridership with machine learning models.
